# 多团队会议记录汇总分析 — 使用指南

> 本指南面向日常负责会议记录整理、报告编写的行政和项目管理人员。
> 无需编程基础，按步骤操作即可完成。

---

## 一句话说明

把散落在金山文档（KDocs）里的几十、上百份会议记录，**自动读取、自动分析、自动生成 Word 报告和可视化看板**——原来需要 1-2 天手工整理的工作，现在几分钟搞定。

---

## 这个工具能做什么

### 功能一：自动读取会议文档

- 连接金山文档云盘，按你指定的**日期范围**自动筛选会议记录
- 同时读取多份文档（5 路并发），速度是逐个打开的 5 倍
- 自动识别文档中的**结论、待办事项、参会人员**

> 举例：你输入「03-10 到 04-30」，系统会自动找出这段时间内所有团队的会议记录并读取。

### 功能二：生成 Word 分析报告

系统会生成两种报告，均为标准 `.docx` 格式，可直接用 Word/WPS 打开编辑：

| 报告类型 | 说明 | 适合场景 |
|----------|------|----------|
| **单团队报告** | 每个团队一份独立报告 | 给各团队负责人看 |
| **综合分析报告** | 所有团队汇总，跨部门视角 | 给领导汇报、全局复盘 |

**报告内容包括 6 大板块：**

```
一、执行摘要          ← 核心数据一目了然（会议数量、覆盖部门等）
二、风险点分析        ← 高/中风险自动标红标橙，附风险矩阵表格
三、重点关注节点      ← 近期和中期的关键时间节点，标注责任方和来源
四、各部门会议汇总    ← 每个团队的核心议题和关键决议
五、综合评估与建议    ← 整体概况分析 + 行动建议
六、附录              ← 完整会议清单和文档纳入率统计
```

报告自带**专业排版**——封面、页眉页脚、蓝色主题配色、表格样式，无需额外美化，**打印或投屏都好看**。

### 功能三：生成会议看板（HTML 页面）

生成一个可以在浏览器中打开的**交互式看板页面**：

- **矩阵视图**：横轴是周次，纵轴是团队，一眼看清谁在什么时候开了什么会
- **点击跳转**：每个会议标题可直接点击，跳转到金山文档原文
- **重要标记**：重要人员参加的会议自动标红 + 显示【重要】标签
- **筛选搜索**：支持按「全部 / 含重要 / 有会议」筛选，支持关键词搜索

> 看板特别适合**项目管理、行政助理**日常掌握各团队会议动态。

## 核心优势

### 省时间

| 传统方式 | 使用本工具 |
|----------|-----------|
| 手动逐个打开几十份文档 | 一条命令自动读取全部文档 |
| 逐份阅读、手工摘录要点 | 自动提取结论、待办、风险 |
| 在 Word 里排版 1-2 天 | 几分钟生成带封面的专业报告 |
| 手动制作会议统计表格 | 自动生成可交互的会议看板 |

### 不遗漏

- **三重兜底提取**：即使文档格式不统一，系统也能通过多种策略提取信息，确保不漏
- **全量覆盖**：自动扫描所有团队、所有月份文件夹，不会因为「忘了看某个文件夹」而遗漏

### 质量有保障

- **内容去重**：相似度超过 60% 的重复内容自动合并，不会同一件事说两遍
- **风险分级**：同一事项只保留最高风险等级，不会高中风险重复出现
- **文本清洗**：自动去除文档中的序号前缀、勾选框符号等原始标记
- **无效过滤**：纯标签（如「待定」「TBD」）、空洞短语（如「后续讨论」）自动过滤

### 格式统一

不管有多少个团队、多少份文档，生成的报告格式**完全统一**——封面风格、标题层级、表格配色、风险标注颜色全部一致，适合正式场合使用。

## 使用前的准备

### 你需要准备什么

| 准备项 | 说明 |
|--------|------|
| 电脑环境 | 已安装 Node.js 18 以上版本 |
| 金山文档 | 会议记录存放在金山文档（KDocs）的共享文件夹中 |
| 文件夹链接 | 每个团队的金山文档文件夹链接（类似 `https://365.kdocs.cn/folder/xxxxx`） |
| 团队信息 | 团队名称、对应的文件夹链接 |

### 文件夹结构建议

金山文档中的会议记录建议按以下方式组织：

```
团队A 文件夹/
├── 3月/
│   ├── 0301-项目周会
│   ├── 0308-项目周会
│   └── ...
├── 4月/
│   ├── 0401-项目周会
│   └── ...
└── 5月/
    └── ...
```

> 文档标题中需包含日期信息（如 `0301`、`03-01` 等），系统靠标题日期来筛选。

## 使用方法（4 步完成）

### 第 1 步：初始化配置（只需做一次）

运行配置向导，按提示输入团队名称和文件夹链接：

```bash
node scripts/setup.js
```

系统会引导你：
1. 输入团队名称（如：技术部、产品部）
2. 粘贴金山文档文件夹链接
3. 自动解析并保存到配置文件

配置完成后，信息保存在 `config.json` 中。**以后不用再配置，直接从第 2 步开始。**

> 如果后续需要新增团队，重新运行此命令或手动编辑 `config.json` 即可。

### 第 2 步：批量读取文档

指定日期范围，自动读取所有团队的会议记录：

```bash
node scripts/batch-read-documents.js 起始日期 结束日期
```

**示例：**

```bash
# 读取 4 月 1 日到 4 月 30 日的会议记录
node scripts/batch-read-documents.js 04-01 04-30

# 读取 3 月 10 日到 5 月 30 日的会议记录
node scripts/batch-read-documents.js 03-10 05-30
```

运行后系统会显示进度，完成后在 `outputs/` 中生成数据文件 `all-team-summaries.json`。

### 第 3 步：生成 Word 报告

```bash
# 先生成各团队独立报告
node scripts/generate-team-report.js 04-01 04-30

# 再生成综合分析报告
node scripts/generate-comprehensive-report.js 04-01 04-30
```

> 日期范围要和第 2 步一致。

完成后，`outputs/` 文件夹中会出现：
- `{团队名}-会议记录汇总分析报告-0401-0430.docx`（每个团队一份）
- `SEED-会议记录汇总分析报告-0401-0430.docx`（综合报告）

**直接双击用 Word 或 WPS 打开即可。**

### 第 4 步：生成会议看板（可选）

```bash
node scripts/generate-kanban.js
```

生成 `outputs/会议看板.html` 文件，双击用浏览器打开即可查看和操作。

> 看板会在第 3 步生成团队报告后**自动附带生成**。如果只想单独更新看板，手动运行上面的命令即可。

## 常见问题

### Q：日期范围怎么写？

使用 `月-日` 格式，如 `03-10` 表示 3 月 10 日，`04-30` 表示 4 月 30 日。起止日期之间用空格分开。

### Q：新增了一个团队怎么办？

重新运行 `node scripts/setup.js`，或在 `config.json` 文件中手动添加团队配置。

### Q：新的月份（比如 6 月）开始了怎么办？

在 `config.json` 中对应团队的 `months` 字段里添加新月份的文件夹 ID。

### Q：文档标题没有日期怎么办？

系统根据文档标题中的日期来筛选。如果标题没有日期信息（如 `0501` 或 `05-01`），该文档可能不会被纳入。建议文档标题统一包含日期。

### Q：报告中某些内容不准确怎么办？

报告生成后是标准 Word 文件，可以直接打开编辑修改。系统保证**不编造内容**，所有结论均来自原始会议记录。

### Q：没有网络可以用吗？

如果之前读取过文档并有缓存数据，可以运行 `node scripts/reconstruct-from-cache.js` 从本地缓存重建数据，然后生成报告。

### Q：看板打开是空白的？

确保已经执行过第 2 步（读取文档），看板需要 `outputs/all-team-summaries.json` 数据文件才能生成内容。历史根目录数据仍会被兼容读取。

## 输出文件一览

| 文件 | 说明 | 怎么打开 |
|------|------|----------|
| `outputs/all-team-summaries.json` | 所有团队会议数据（中间文件） | 不需要打开，给程序用的 |
| `outputs/team-summary-{团队名}.json` | 各团队独立数据（中间文件） | 不需要打开，给程序用的 |
| `outputs/{团队名}-会议记录汇总分析报告-xxxx.docx` | 单团队 Word 报告 | Word / WPS |
| `outputs/综合分析报告-xxxx.docx` | 综合分析 Word 报告 | Word / WPS |
| `outputs/会议看板.html` | 交互式会议看板 | 浏览器（Chrome / Edge） |
| `outputs/会议看板-data.json` | 看板数据（中间文件） | 不需要打开，给程序用的 |

> 标注「中间文件」的是程序运行产生的数据文件，你只需要关注 `.docx` 和 `.html` 文件即可。
